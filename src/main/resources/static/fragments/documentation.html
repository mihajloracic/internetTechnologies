INTRODUCTION

 

Web application is intended for people how are planning to go out and need system to pick a place where to go out. Application will consist of administrator part how adds places (clubs, pubs, bars etc…), basic users how create group for going out with their friends, and vote for their favorite places to help them decide where to go out.

Importance and Relevance

Application like this doesn’t exist on the market and we believe it might prove useful to lots of users. Academic aspect of our application is to test different systems of decision making (voting). For example: majoritarian systems vs alternative vote.

Goal and Objectives

Goal of this project is to help users plan all aspects of event:

  1. Browse available places and events
  2. Read and comment/rate on places and events
  3. Create group with their friends
  4. Systematically vote on where to go with in their group

In this section we will compare two similar systems for our targeted users and how Trip Advisor and Night Out how to they stand to one another.

1.1.   Similar system analysis

1.1.1.  Night Out

 

Night Out is an online tool to compare events, places to go out based on the city chosen by the user. The Google Page rank of the website is 6/10 (checkpagerank.net), probably the best night out comparator on internet.

 
   

 

 

Figure 1-1 Night out comparator

Advantages:

  • Simple user interface
  • Compare only night out places (not restaurants, hostels, )

 

Disadvantages:

  • Only criteria asked : the city to go out
  • Based only in US

 

1.1.2. TripAdvisor

 

Easy price comparison · Millions of hotel reviews · Candid traveler photos

TripAdvisor is a places comparator providing hostels, restaurants, flight tickets, etc. It’s maybe the best places comparator on internet with a Google Page rank of 7/10 (checkpagerank.net) thanks to its lot of data and its easy way to use.

 
   

 

 

 

 

Figure 1-2 TripAdvisor

Advantages:

  • Simple user interface
  • Compare a lot of kind of places (restaurants, hostels, flight tickets, )
  • Have a big database and million of reviews

 

Disadvantages:

  • Does not compare events
  • For our user group most of data is irrelevant, for example airplane tickets, restaurants, hotels

2.1.   Functional requirements

There is any possibilities that what can the user do. First, he needs to be logged in or register if he doesn‘t have an account. The admin, can add a place, modify a place and/or delete it. But to be done, the super admin has to confirm the new place added and the modifications that has be done to the place by the admin. The user can search a place, can also rate it, vote it with their friends to know where they‘ll night out and comment it after visited it. So he also can invite his friends to vote for a place.

Users must be able to choose how who can see their profile and posts. Passwords should be stored as SHA1 hashes.Connections should be encrypted to prevent unauthorised listening of communication.

Using the Maven Plugin

The Spring Boot Maven plugin includes a run goal that can be used to quickly compile and run your application. Applications run in an exploded form, as they do in your IDE. The following example shows a typical Maven command to run a Spring Boot application:

$  mvn  spring-boot:run                                                                                                                            

 

You might also want to use the MAVEN_OPTS operating system environment variable, as shown in the following example:

$  export  MAVEN_OPTS=-Xmx1024m                                                                                                                

 

To change port on which application runs change .port property in properties.yml file.

Plan Night Out is doing something that nothing else can do or have been invented  before. It‘s similar to  Night out comparator (Figure 1-1) but we added some features like you can vote for a place or invite some friends to vote and rate any place. You can also choose what type of establishment you want to go (restaurant, bar, pub, club, etc.)  and we can‘t find something similar on Internet.

 

As Wenjia Jiang said in her final urban and economic geography master thesis report about „The role of urban nightlife in perceived social integration“, she said „Nightlife is about meeting others, creating identities and having fun, as well as seeking out intercultural, interracial and cross-gender sociability in global nightscapes“ and that‘s what we try to do with our web application : let the people meeting each other in the nightlife places and having fun. There is some ritual that students are doing  oftently is to pre-drinking somewhere (dormitory or friend‘s flat before to go out but all are not agreed about the place to go and Plan Night out is figuring out the place to go.
